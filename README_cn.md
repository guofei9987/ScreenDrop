# SreenDrop

> 基于二维码的离线文件传输工具。
>
> 无需网络，无需蓝牙，无需安装应用。

SreenDrop 是一个运行在浏览器中的文件传输工具，通过播放二维码序列，在不同设备之间传输文件。

你只需要：

* 一块用于显示二维码的屏幕；
* 一个带摄像头的接收设备。

无需 Wi-Fi、无需蓝牙、无需 AirDrop，也无需安装任何原生 App。

---

## ✨ 功能特性

* 📶 **完全离线**：不依赖任何网络连接；
* 🌐 **浏览器运行**：无需安装软件；
* 📷 **屏幕 + 摄像头即可使用**；
* 📦 **支持大文件传输**：自动切分为多个二维码分片；
* 🔒 **SHA-256 完整性校验**：确保文件内容正确；
* 🔄 **循环播放机制**：提高传输成功率；
* 🧩 **缺失分片检测与补传**：支持恢复遗漏的二维码；
* 📱 **跨平台**：支持手机、平板、电脑等设备。

## 📖 使用说明

### 发送文件

1. 打开 `sender.html`；
2. 选择需要发送的文件；
3. 根据需要调整传输设置；
4. 点击 **开始发送**；
5. 将二维码展示给接收设备扫描。

### 接收文件

1. 打开接收页面；
2. 允许浏览器使用摄像头；
3. 将摄像头对准发送端屏幕；
4. 等待所有分片接收完成；
5. 下载恢复后的文件。

---

## ⚙️ 传输模式

SreenDrop 提供多种分片大小档位：

| 模式       | 说明              |
| -------- | --------------- |
| 兼容模式     | 二维码密度较低，更容易识别   |
| 平衡模式（推荐） | 在速度与可靠性之间取得最佳平衡 |
| 极速模式     | 二维码密度较高，传输速度更快  |

分片越大，传输速度通常越快，但在部分老旧设备上的识别成功率可能下降。

---

## 📋 使用建议

* 推荐传输文件大小：≤ 20 MB；
* 摄像头清晰度越高，传输体验越好；
* 建议在光线充足的环境下使用；
* 推荐使用现代浏览器：

  * Chrome
  * Edge
  * Safari

## ❓ 常见问题

### 为什么不使用蓝牙？

蓝牙配对过程相对繁琐，且不同平台之间兼容性有限。

### 为什么不使用 AirDrop？

AirDrop 仅适用于苹果生态设备。

### SreenDrop 传输速度快吗？

SreenDrop 的设计目标是**通用性和离线可用性**，而非极致速度。

### 如果漏扫了一些二维码怎么办？

接收端会自动识别缺失的分片编号，发送端可直接跳转到对应分片进行补传。

---

## 🔒 数据完整性

每次传输都会进行完整性校验，包括：

* 分片级 CRC 校验；
* 文件级 SHA-256 校验。

如果文件内容发生损坏，系统会自动检测并拒绝生成错误文件。

---

## 🧠 工作原理

SreenDrop 的传输流程如下：

```
文件
 ↓
切分为多个分片
 ↓
将每个分片编码为二维码
 ↓
循环播放二维码序列
 ↓
接收端扫描二维码
 ↓
重组文件
 ↓
进行 SHA-256 校验
 ↓
下载文件
```
